## 3. 各功能详细说明

### 3.1 项目管理

#### 创建项目对话框

**触发**: 点击首页 "+" 卡片

**表单字段**:
| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 书名 | text input | ✅ | — | 2-50字符 |
| 类型 | select | ✅ | 玄幻 | 18种类型 |
| 受众 | radio | ✅ | 通用 | 男频/女频/通用 |
| 核心创意 | textarea | ✅ | — | 一句话描述核心创意 |

**行为**:
1. 点击 "创建" → 写入 IndexedDB → 跳转到工作台
2. 自动创建关联的空 Worldview、StoryCore 等记录
3. 默认进入 "基本信息" 面板

#### 项目卡片

**显示信息**:
- 书名
- 类型标签
- 章节数 / 总字数
- 最后编辑时间
- 创建时间

**操作**:
- 点击 → 进入工作台
- 右键/更多按钮 → 删除（二次确认）

### 3.3 章节写作编辑器

#### 编辑器选型

**推荐**: TipTap（基于 ProseMirror）

**理由**:
- 支持 Markdown 快捷输入
- 可扩展性强（可添加自定义节点）
- 支持协同编辑（未来扩展）
- React 集成良好
- 内置丰富的格式支持

**降级方案**: 纯 textarea + Markdown 预览（MVP 阶段可先用这个）

#### AI 操作说明

| 操作 | 触发条件 | 行为 |
|------|---------|------|
| **生成正文** | 章节内容为空 | 从大纲+世界观生成完整章节 |
| **续写** | 章节有内容 | 从当前内容末尾继续写 |
| **扩写** | 选中一段文字 | 将选中内容扩展为更丰富的段落 |
| **润色** | 选中一段文字 | 优化文字表达，使之更流畅 |
| **去 AI 味** | 选中一段文字 | 消除 AI 生成的典型痕迹 |
| **自定义指令** | 任何时候 | 用户输入自定义指令让 AI 处理 |

### 3.4 大纲树

#### 交互设计

```
📖 大纲树
│
├── [+添加卷]  [🤖AI生成整体大纲]
│
├── 📁 第一卷：初入江湖  [✏️] [🤖] [🗑] [⬆⬇]
│   ├── [+添加章节]  [🤖AI展开本卷]
│   ├── 📄 第1章：老人与少年  [✏️] [🗑] [⬆⬇]
│   │   └── 摘要：老者在山间遇到少年，赠剑...
│   ├── 📄 第2章：古剑入手  [✏️] [🗑] [⬆⬇]
│   │   └── 摘要：少年发现古剑的异常...
│   └── ...
│
├── 📁 第二卷：宗门试炼  [✏️] [🤖] [🗑] [⬆⬇]
│   └── (尚未展开)  [🤖AI展开本卷]
│
└── ...
```

**操作说明**:
- ✏️ = 编辑节点标题/摘要
- 🤖 = AI 重写/重新生成
- 🗑 = 删除节点（含子节点）
- ⬆⬇ = 拖拽排序
- AI 生成整体大纲 = 生成所有卷级节点
- AI 展开本卷 = 生成该卷下的章节节点


#### 伏笔状态流转

```
 ┌──────┐    埋设    ┌──────┐    呼应    ┌──────┐    回收    ┌──────┐
 │已计划│ ─────────→ │已埋设│ ─────────→ │已呼应│ ─────────→ │已回收│
 │planned│           │planted│           │echoed│           │resolved│
 └──────┘           └──────┘           └──────┘           └──────┘
     │                                                        
     │              放弃                                      
     └──────────────────────────────→ ┌──────────┐          
                                       │已放弃    │          
                                       │abandoned │          
                                       └──────────┘          
```

每次状态变更需关联章节：
- 埋设 → 指定在哪一章埋设 + 描述埋设方式
- 呼应 → 指定在哪一章呼应 + 描述呼应方式（可多次）
- 回收 → 指定在哪一章回收 + 描述回收方式

### 3.6 数据导入导出

#### JSON 导出

**触发**: 工作台 → 更多 → 导出为 JSON

**行为**:
1. 从 IndexedDB 读取当前项目所有数据
2. 组装为 `StoryForgeBackup` 结构
3. 生成 `{书名}_backup_{日期}.json` 文件
4. 触发浏览器下载

#### JSON 导入

**触发**: 首页 → 导入项目

**行为**:
1. 选择 `.json` 文件
2. 校验文件格式（version、必填字段）
3. 预览项目信息（书名、章节数等）
4. 确认导入
5. 写入 IndexedDB（生成新的 projectId，避免冲突）

#### Markdown 导出

**触发**: 工作台 → 导出 → Markdown

**行为**:
1. 按大纲结构组织章节
2. 只导出正文（不含世界观设定）
3. 生成 `{书名}.md` 文件

### 4.1 性能
