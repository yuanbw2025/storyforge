## 一、产品定位与技术架构

### 1.1 是什么

StoryForge 是一个**纯浏览器端的 AI 辅助长篇小说创作工具**。用户在浏览器里完成从世界观构建到章节写作的全流程，AI 在每个环节提供辅助生成，但最终决策权始终在作者手中。

### 1.2 核心设计理念

- **纯前端，零后端**：所有数据存在浏览器 IndexedDB 中，不经过任何服务器。用户的创作内容只存在于自己的设备上。
- **AI 是助手，不是代笔**：每次 AI 生成的结果都以"预览→确认→编辑"的方式呈现，用户可以采纳、修改或丢弃。
- **全链路上下文感知**：写正文时，AI 自动读取世界观、角色设定、大纲、伏笔状态、创作规则，确保生成内容与已有设定一致。
- **提示词完全可定制**：所有 AI 调用的 prompt 都在「提示词库」中公开，用户可以查看、修改、替换。

### 1.3 技术栈

| 层 | 技术 |
|---|---|
| 框架 | React 18 + TypeScript + Vite |
| 状态管理 | Zustand（27 个 Store） |
| 本地数据库 | Dexie.js（IndexedDB 封装），42 张表（DB v24） |
| AI 调用 | OpenAI 兼容协议，流式 SSE |
| UI | Tailwind CSS + Lucide Icons |
| 部署 | Vercel（静态站点），支持 PWA 离线 |

### 1.4 兼容的 AI 平台（19 家，含 NVIDIA NIM）

| 平台 | 默认模型 | 特点 |
|---|---|---|
| DeepSeek | deepseek-v4-flash | 国产首选，性价比极高 |
| 通义千问（Qwen） | qwen-max | 阿里云，中文理解强 |
| 豆包（Doubao） | doubao-pro-32k | 字节跳动，适合对话 |
| MiniMax | MiniMax-Text-01 | 245K 超长上下文 |
| 智谱（GLM） | glm-4-flash | 免费额度，适合试用 |
| 文心一言（Wenxin） | ernie-4.0-8k | 百度，中文基础好 |
| Gemini | gemini-2.5-flash | Google，100 万上下文窗口 |
| Poe | GPT-4o | 聚合平台 |
| OpenAI | gpt-4o | 英文最强 |

## 二、项目首页

### 2.1 项目列表

打开 StoryForge 后看到的第一个页面。展示所有创建的小说项目，按最近更新时间排序。

**每个项目卡片显示**：
- 项目名称 + 首字符装饰图标（自动生成的彩色圆形）
- 流派标签（如"仙侠""言情"）
- 写作状态标签（构思中 / 撰写中 / 修改中 / 已完结）
- 目标字数
- 最近更新时间

### 2.2 创建项目

点击「+ 新建项目」弹出创建对话框：

- **项目名称**：必填
- **简介**：一句话描述故事
- **流派**：多选标签，内置 12 个常用流派（仙侠、言情、悬疑推理、科幻、历史、都市、玄幻、武侠、末世、重生、系统流、西幻），支持自定义流派
- **目标字数**：滑动条选择（10 万 ~ 500 万）
- **多世界模式**：可选开关（默认关），开启后一个项目可管理多个独立世界（见第十九章）

> **真实与幻想（Phase 32）**：早期的「幻想 / 历史考证」二选一已废弃，改为进入项目后在世界观「⚖️ 真实与幻想」面板做**维度级**约束声明——可在任意维度自由混搭真实与架空，AI 生成时按声明遵守（见 4.0）。

## 三、著作信息

### 3.1 项目概况

项目的"身份证"页面，可随时修改：

- 项目名称、简介
- 流派多选标签（与创建时同款）
- 目标字数
- 写作状态切换
- 创作模式切换（幻想 ↔ 历史考证）

### 3.2 项目参考

管理与本项目相关的参考资料。支持三种类型：

| 类型 | 用途 | 图标颜色 |
|---|---|---|
| 故事参考 | 剧情结构、人物塑造的范本 | 蓝色 |
| 风格参考 | 文笔风格、叙事技巧的范本 | 紫色 |
| 历史资料 | 历史考证用的文献 | 琥珀色 |

**每个参考书的信息**：
- 书名、作者、类型
- 可手动录入关键笔记
- 支持 AI 导入解析（上传 TXT 文件 → AI 自动提取世界观/角色/大纲）

#### 3.2.1 深度分析

参考书的核心功能——**十三维 AI 深度分析**：

上传参考作品的原文（TXT），系统按设定的深度（快速/标准/精细）分块，逐块调用 AI 做多维分析：

**通用八维**：
1. 叙事架构
2. 开篇与黄金三章
3. 情节结构与节奏
4. 人物塑造
5. 冲突与升级
6. 伏笔与悬念
7. 文笔与对话
8. 世界观构建

**历史资料专属五维**（仅历史类型参考书显示）：
9. 历史背景与时代特征
10. 社会制度与等级
11. 日常生活细节
12. 物质文化（器物/科技）
13. 语言习惯与称谓

分析结果按维度 Tab 展示，每个维度下有多个分块的分析条目。

> **联动逻辑**：分析结果不是孤立的——在「创作规则」面板中，用户可以勾选参考书，被勾选的参考书的分析结论会被自动注入 AI 写作的上下文中。相当于告诉 AI："写的时候参考这本书的手法。"

#### 3.2.2 作品学习（大师研究）

入口整合在「项目参考」面板中，但功能独立。上传已完成的名家作品原文进行深度拆解。

**三层分析架构**：

| 层级 | 内容 | 方式 |
|---|---|---|
| Layer 1：五维方法论 | 世界观范式、角色设计手法、情节节奏规律、伏笔悬念、文笔语言 | AI 逐块分析 |
| Layer 2：深度量化 | 风格画像（句长分布、对话占比、高频词、段落密度）+ 章节节奏点时间线（开场/冲突/反转/高潮/钩子/伏笔/松弛） | 风格=本地计算；节奏=AI |
| Layer 3：跨作品洞察 | 多本作品的共性方法论归纳（如"猫腻式插叙节奏法则"） | AI 归纳 |

**特色功能**：
- **Blob 持久化**：上传的原文 Blob 存入 IndexedDB，关闭浏览器后重开可续跑未完成的分析
- **断点续传**：如果分析中途断了（网络/取消），下次从断点继续，已分析的块不会重复调用
- **分析专用模型**：可在学习设置中单独配置低成本模型（如 GLM-4 Flash 免费），不影响全局写作模型
- **费用预估**：上传文件后、分析前，展示预估 Token 消耗和费用
- **导出 ZIP 报告**：分析结果可导出为 ZIP（含 JSON 数据 + 人读的 Markdown 报告）

> **联动逻辑**：Layer 3 生成的洞察可在「创作规则」面板中被勾选引用，AI 写作时自动注入方法论要点。

## 四、设定库 — 世界观

世界观分为三个子面板，各有独立入口，共同构成完整的世界设定。

> **架构升级（Phase 32）**：早期的「幻想模式 / 历史考证模式」二选一已废弃，改为**维度级的「真实与幻想」规则体系**（见 4.0）——可在任意维度上自由混搭真实与架空。下文各面板字段为统一标签，不再区分两种模式。
>
> **多世界（Phase 25.4，见第十九章）**：开启多世界模式后，世界观三面板、力量体系、地理、世界地图面板顶部出现**世界切换器**，切换后编辑对应世界的独立设定；AI 生成时仅读取当前世界的上下文，互不混淆。

### 4.0 真实与幻想（世界规则体系）

世界观子树第一项「⚖️ 真实与幻想」。将历史考证从「项目级二选一」改为「**维度级约束声明**」：

- 三级树结构（15 大类 → ~50 子类 → 提示标签）：时代背景 / 重大事件 / 地理疆域 / 气候环境 / 政治制度 / 军事 / 经济 / 社会结构 / 科技生产力 / 文化思想 / 宗教信仰 / 民族族群 / 语言称谓 / 日常生活 / 力量与超自然
- 每个节点可独立设为：**📜 取自真实 / ✨ 架空改造 / ⚖️ 冲突优先**
- 用户可自定义新增 L1/L2/L3 节点
- 底部规则清单实时预览 + Token 估算

> **联动逻辑**：`buildWorldRulesContext()` 把规则清单注入下游所有 AI 生成（世界观各维度、大纲、正文、场景考证）。标注「取自真实」的维度按真实历史考据、杜绝时代错乱；标注「架空改造」的维度尊重用户设定不被否定。与历史年表的「⚓ 史实锚点」事件联动——锚点事件 AI 不可违反。

### 4.1 世界起源

字段：世界来源（创世神话/历史起源）、力量层次（修真等级/社会阶层/科技层级，含晋升规则）、神明与信仰（层级/名号职司/规则禁忌）。

每个字段都支持**单字段 AI 生成**：点击字段旁的 ✨ 按钮，AI 根据已有的其他字段内容生成该字段。生成结果以流式方式展示（打字机效果），用户可实时看到输出，满意后点击采纳。

> **小巧思**：AI 生成某字段时，会自动读取**世界观三面板互通的关键字段**作为上下文（世界起源 ↔ 自然环境 ↔ 人文环境互相参考），确保设定自洽、不割裂。
>
> **AI 拆分（Phase 26 修复）**：「神明与信仰」生成后采纳时，由 AI 二次解析自动拆分为「信仰层级 / 名号职司 / 规则禁忌」三个子字段（不用正则，准确率高）。

### 4.2 自然环境

统一字段：世界结构（星球/大陆/多重天）、疆域尺寸、地貌分布、重镇分布、山川水系、气候环境、自然资源（珍禽异兽/灵药/矿物/其他）。逐字段 AI 生成，与世界起源、人文环境互通上下文。

> **导航提示**：地貌分布 / 重镇分布字段下方提示「概述写这里，可视化地图与具体地点请到『🗺️ 世界地图』管理」。

### 4.3 人文环境

统一字段：世界历史线、世界大事记、种族与民族、势力分布（含内嵌**势力管理面板**，详见 7.5）、政治/经济/文化、矛盾冲突、道具与器物。另含**货币管理面板**（多种货币 CRUD + 兑换关系）。

> **导航提示**：世界历史线/大事记 → 提示前往「📜 历史年表」管理具体事件；道具与器物 → 提示前往「💎 道具系统」/「🎒 物品栏」。
>
> **联动逻辑**：货币面板创建的货币体系会被 `buildWorldContext()` 读取，AI 生成章节时自动注入，确保金钱描写符合设定。

> **联动逻辑**：货币面板中创建的货币体系会被 `buildWorldContext()` 读取，在 AI 生成章节时自动注入，确保小说中的金钱描写符合设定。

### 4.4 历史年表

独立入口，管理世界的历史事件和时代关键词。

**三个 Tab**：
1. **时间线**：事件 CRUD，每个事件有年份、名称、描述、时代分类，支持排序
2. **关键词**：按类别（人物/事件/地理/文化/制度/器物/其他）管理历史关键词，每个关键词有名称、释义、时代标签
3. **总览**：历史概述 + 纪年体系说明的文本编辑

支持 AI 生成事件详情和关键词释义。事件分「⚓ 史实锚点（AI 不可违反）」与「✨ 虚构/架空」两类。

**多世界（Phase 25.5.1）**：开启多世界模式后，历史年表顶部出现**一排世界标签**（主世界 | 世界A | 世界B | … | 📋 一览）：
- 选具体世界标签：概述/时间线/关键词整体作用于该世界，新建项自动归属该世界
- 「一览」标签：只读合并视图，所有世界事件按时间排列并标注所属世界，便于横向对比

> **联动逻辑**：历史年表的事件和关键词会被 `buildHistoricalContext()` 读取，在大纲和正文生成时注入 AI 上下文（限制 2000 字符以内，避免占用过多 Token）。AI 历史考证/头脑风暴会注入当前世界的世界观设定。

### 4.5 世界地图

可视化的世界地图系统，支持三种渲染模式：

1. **Voronoi 地图**（默认）：程序化生成的多边形地图，支持 AI 一键生成配置 → 随机生成地图 → 手动调整
2. **2D Canvas 地图**：简单的 2D 渲染
3. **3D 地图**：三维球体渲染（实验性）

**世界树侧边栏**：
- 左侧展示树状的世界节点列表
- 支持多层级（主世界 → 子位面 → 子区域）
- 每个世界节点可有独立的地图配置
- 世界之间可添加**传送门**（Portal），记录名称、坐标、目标世界

**AI 地图生成**：
- 点击「AI 生成」按钮，AI 根据世界观设定生成地图配置参数（大陆数量、海洋比例、气候分布等）
- 配置自动填入 Voronoi 引擎，生成随机地图
- 用户可多次重新生成直到满意
- 解析失败时显示明确错误提示（Phase 26 修复，不再静默卡住）

**多世界（Phase 25.4）**：开启多世界模式后，世界地图按世界组隔离——每个世界拥有独立的世界树与地图，顶部世界切换器切换时整套地图随之切换；AI 地图生成读取**当前世界**的地理设定（地貌/山川/气候），不会用别的世界的设定生成本世界的地图。

> **小巧思**：地图不是纯装饰——生成的地图配置存储在 WorldNode 的 `mapConfigJSON` 字段中，地图数据缓存在 `mapCacheJSON` 中，切换世界节点时地图会跟着切换。世界树节点通过 `worldGroupId` 隶属于世界组（见第十九章）。

## 五、设定库 — 故事设计

对应侧边栏「故事设计」入口，管理故事的核心设定。

**字段列表**：
- 核心主题
- 故事类型
- 核心冲突
- 故事主线
- 情感基调
- 写作风格
- 节奏设计

每个字段支持 AI 单字段生成。AI 生成时会读取已有的世界观摘要作为上下文——确保故事设计与世界观一致。

> **联动逻辑**：故事设计中的「核心冲突」和「故事主线」会被注入到大纲生成的 prompt 中，引导 AI 按主线展开卷/章结构。

## 六、设定库 — 道具系统

管理小说中的重要物品、法宝、装备。

**每个道具的字段**：
- 名称
- 类别（自定义）
- 等级/品阶（自定义）
- 外观描述
- 能力效果
- 来历/出处
- 当前持有者

支持折叠展开详情、增删改查。

> **与「物品栏」的分工（Phase 25.5.2-b）**：道具系统是**手动管理**的设定库（写世界里有哪些重要物品）；而创作区的「🎒 物品栏」是**AI 从已写正文自动提取**的游戏包裹式追踪（主角实际获得/消耗了什么，含数量与历程，详见 21.1）。一个是设定，一个是产物。

## 七、设定库 — 角色设计

角色系统分为四个子面板，按角色重要性分级管理。

### 7.1 主要角色

管理主角、反派和重要配角。

**创建角色时**：
- 弹出角色类型选择器（6 种），带图标引导：
  - 主角（protagonist）
  - 反派（antagonist）
  - 重要配角（supporting）
  - 次要角色（minor）
  - NPC
  - 路人（extra）
- 选择后创建角色卡

**角色卡字段**：
- 基础：姓名、性别、年龄、身份/职业
- 外貌描述
- 性格特征
- 背景故事
- 动机与目标
- 能力/技能
- 人物弧光（成长变化）
- 口头禅/说话风格
- 关系描述
- 备注

**AI 角色生成**：
- 点击「AI 生成角色」，输入提示词（如"帮我生成一个反派"）
- AI 生成时会自动感知当前角色阵容缺口：如果项目没有主角，AI 会优先建议生成主角
- 生成结果自动解析为角色卡，用户确认后入库

**角色动态状态面板**：
- 角色详情底部显示该角色在状态表中的最新状态
- 从 StateCard 表中读取与该角色关联的状态卡，展示关键字段变化

**多世界归属（Phase 25.4）**：开启多世界模式后，角色面板顶部出现世界过滤器（全部 / 各世界 / 🌐 跨世界）；角色详情卡可设置归属世界或标记为「跨世界」（如主角、系统精灵等在所有世界可见）。AI 写作时按所属世界过滤角色上下文。

### 7.2 次要角色 / 7.3 NPC / 7.4 路人

三个子面板结构相同，按角色类型自动过滤。
- 次要角色面板只显示 `role: 'minor'` 的角色
- NPC 面板只显示 `role: 'npc'` 的角色
- 路人面板只显示 `role: 'extra'` 的角色

### 7.5 关系网

管理角色之间的关系。

**两种视图**：
1. **关系图**（默认）：基于 react-force-graph-2d 的力导向图，每个角色是一个节点，关系是连线。节点大小按角色重要性（role）自动调整。
2. **列表视图**：传统表格，展示所有关系条目

**每条关系的字段**：
- 角色 A → 角色 B
- 关系类型（师徒/恋人/敌对/同门/家族/同盟等）
- 关系描述
- 双向/单向

> **联动逻辑**：角色数据在 AI 生成正文时按三层权重注入——主角/反派输出完整性格和动机（约 200 字），重要配角输出一句话简介+关系，NPC/路人仅列名字。这样既保证 AI 了解核心角色，又不会因为角色太多撑爆上下文。

## 八、创作区 — 创作规则

控制 AI 写作行为的"红线"面板。

**设置项**：
- **写作风格**：自由文本描述（如"简洁明快，多用短句"）
- **叙事视角**：下拉选择（第一人称 / 第三人称有限 / 第三人称全知 / 多视角）
- **基调与氛围**：自由文本
- **写作禁区**：标签列表（如"主角不能死""不准出现现代用语"），AI 写作时作为硬约束
- **一致性规则**：标签列表（如"同一场景不能超过 3 个角色同时对话"）
- **特殊要求**：自由文本
- **参考作品引用**：下拉多选，可勾选「项目参考」中已分析的作品

**AI 生成规则**：点击按钮，AI 根据已有的世界观、故事设计、角色设定，自动建议一套创作规则。

> **联动逻辑**：创作规则中的**全部内容**都会在 AI 生成章节正文时被注入到 system prompt 中。这是 StoryForge 保持生成一致性的核心机制——不管用户调用哪个生成功能，创作规则都是 AI 的"行为准则"。

**引用参考作品分析**：勾选后，该参考书的十三维分析结论会被精炼后注入 AI 上下文。

**引用大师洞察**：勾选后，作品学习系统归纳的方法论要点会被注入 AI 上下文。

## 九、创作区 — 大纲

大纲系统采用树状层级结构：**卷 → 篇章 → 故事块 → 章节**（支持但不强制全部层级）。

### 9.1 大纲树

左侧展示大纲树，右侧展示选中节点的详情。

**节点操作**：
- 增删改查每个节点
- 拖拽排序（改变顺序）
- 点击卷节点查看该卷下的章节列表

### 9.2 AI 卷级大纲生成

选中模式为「卷级大纲」后：
- 输入补充说明（可选）
- AI 根据**世界观 + 故事核心 + 角色阵容 + 历史年表**（如有）生成卷级大纲
- 输出格式为 JSON 数组（每卷含标题和摘要）
- 生成结果先进入**预览区**，用户可逐卷确认/修改/删除
- 确认后批量写入大纲树

### 9.3 AI 章节大纲生成

选中某一卷后切换到「章节大纲」模式：
- AI 根据该卷的摘要 + 世界观 + 角色，生成该卷下的章节列表
- 同样先预览再确认

### 9.4 批量生成

「一键生成全部章节大纲」按钮：
- 按卷顺序串行调用 AI
- 每生成完一卷，该卷的摘要会作为上下文传给下一卷，保持故事线连贯
- 支持进度条显示 + 中途取消
- 单卷失败不中断，跳过继续

### 9.5 大纲输出格式

AI 被要求以 JSON 数组格式输出（用 ```json 代码块包裹）。解析时采用**JSON 优先 + 正则降级**策略：
1. 先尝试从 ```json 代码块提取 JSON
2. 失败则尝试清理文本后 JSON.parse
3. 再失败则用正则匹配 `**标题**：摘要` 格式

> **小巧思**：这个三层解析策略是为了兼容不同 AI 模型的输出习惯——有些模型严格按 JSON 输出，有些会加多余的说明文字，有些用 Markdown 格式。三层降级确保任何模型都能用。

### 9.6 大纲预览面板（PromptRunPanel）

大纲生成采用统一的 `PromptRunPanel` 组件，支持：
- 系统提示词预览和自定义覆盖
- 用户提示词预览和自定义覆盖
- 参数面板（如目标字数滑动条、基调选择等）
- 生成后的流式输出展示
- Token 消耗显示

## 十一、创作区 — 章节编辑

### 11.1 章节列表

左侧展示大纲树结构（按卷分组），点击章节节点打开右侧编辑器。

**章节状态流转**：
仅有大纲 → 初稿 → 已修改 → 已润色 → 定稿

### 11.2 章节编辑器

核心写作区，包含以下功能：

**基础编辑**：
- 大文本框编辑正文
- 实时字数统计
- 章节摘要（100-200 字，用于后续章节的上下文）
- 作者笔记

**AI 辅助工具栏**（编辑器底部 4 个按钮）：
1. **扩写**：选中段落或全文，AI 丰富细节
2. **润色**：优化文笔、修辞、节奏
3. **去 AI 味**：五维检查（词汇多样性/句式变化/情感层次/叙事留白/细节真实性），改写为更自然的文笔
4. **自定义指令**：输入任意指令让 AI 处理正文

**浮动工具栏**（选中文字时弹出）：
- 润色 / 扩写 / 缩写 / 改写 / 查漏
- 选中一段文字后自动出现，操作精确到段落级别

**上下文预算条**（ContextBudgetBar）：
- 在编辑器底部显示当前 AI 调用的上下文窗口用量
- 格式：`已用 24.8K / 128K (19%)`
- 颜色分级：绿（< 50%）→ 黄（50-80%）→ 红（> 80%）
- 可展开查看各部分占用明细（世界观 / 前文回顾 / 大纲 / 伏笔 / 参考作品）

**AI 正文生成**：
- AI 生成时自动注入的上下文（按 L0-L3 分层，超预算时从低优先级开始裁剪）：
  - L0（必须）：system prompt + 创作规则 + 叙事视角
  - L1（核心）：当前章节大纲摘要 + 前一章正文（全量）
  - L2（增强）：世界观摘要 + 核心角色 + 伏笔状态
  - L3（锦上添花）：参考作品分析 + 大师洞察 + 货币体系 + 历史年表
- 流式输出（打字机效果）
- **Token 实时估算**：生成过程中按字数估算 Token 消耗（中文 ≈ 1.5 token/字），生成完成后切换为 API 返回的精确值

### 11.3 细纲面板

从大纲面板点击「细纲」按钮进入。将章节大纲拆解为具体场景。

**每个场景包含**：
- 场景标题
- 场景目标
- 角色列表
- 场景描述
- 情感走向

**AI 场景拆分**：根据章节大纲摘要，AI 拆分为 3-5 个场景。

**AI 场景增强**：对已有场景进一步丰富细节。

**批量生成细纲**：一键为所有缺细纲的章节生成场景，支持进度条和中途停止。

**大纲-细纲同步检测**：如果章节大纲被修改了，细纲面板顶部会显示黄色警告条「大纲已变更，细纲可能已过时」，提醒用户重新生成或手动更新。

> **小巧思**：同步检测的原理是在细纲中存储了 `lastUsedSummary` 字段——生成细纲时记录当时的大纲摘要快照，后续对比当前大纲摘要，不一致则告警。

### 15.4 AI 设置

- **选择 AI 平台**：下拉选择 19 家平台之一（含 DeepSeek / 智谱 GLM / Kimi / 通义 / 豆包 / MiniMax / 魔搭社区 / NVIDIA NIM / 自定义等）
- **API Key**：填入对应平台的密钥
- **模型名称**：下拉型平台也可**手动输入任意模型名**（列表外型号）
- **Base URL**：自动填入，支持自定义；CORS 受限平台可一键切换本地代理

**API 配置预设（Phase 27）**：可保存多套 API 配置（不同平台/Key/模型），芯片式**一键切换**，支持保存当前为预设、覆盖、重命名、删除。适合在主力模型、备用模型、便宜模型之间快速切换。

> **小巧思**：API Key 存在 localStorage 中，不进入 IndexedDB，不会被导出/备份功能包含，避免泄露。

## 十六、AI 引擎层

### 16.1 统一调用架构

所有 AI 功能共用同一套调用链路：

```
用户操作 → 选择 PromptTemplate → renderPrompt(模板, 变量) → 拼接 messages[]
→ chat(messages, config) → OpenAI 兼容 API → 流式 SSE 响应
→ useAIStream hook 更新 UI → AIStreamOutput 组件渲染
```

### 16.2 上下文自动注入

AI 生成正文时，以下数据被自动收集和注入（按优先级分层）：

| 层级 | 来源 | 内容 |
|---|---|---|
| L0 必须 | 创作规则 | 写作风格、叙事视角、禁区、一致性规则 |
| L1 核心 | 大纲 + 前文 | 当前章节摘要 + 前一章正文/摘要 |
| L2 增强 | 世界观 + 角色 | 世界观摘要、核心角色详情、配角简介、伏笔状态 |
| L3 锦上添花 | 参考/洞察/历史 | 参考作品分析、大师洞察、货币体系、历史年表 |

当总量超过模型上下文窗口的 80% 时，从 L3 开始裁剪。

### 16.3 Token 估算与显示

- **生成中**：按输出字数实时估算（中文 ≈ 1.5 token/字），显示 `≈ ~450 tokens`
- **生成后**：切换为 API 返回的精确值，显示 `Token: ↑8,234 ↓2,156`
- **所有 AI 功能统一显示**，通过 `AIStreamOutput` 共享组件

### 16.4 上下文预算可视化

`ContextBudgetBar` 组件在章节编辑器中显示：
- 进度条 + 百分比
- 各部分占用明细
- 支持 30+ 个模型的上下文窗口预设（从 8K 到 1M）

### 16.5 全局原则：文本解析一律用 AI（不用正则）

确立全局原则——**本工具一切文本分析与内容提取均调用 AI 实现，不使用正则**（正则准确率太低，只适合一般语料清洗）。

- 新增 `restructure.ts` 通用 AI 重构工具：当 AI 输出不是规整 JSON 时，不降级到正则，而是再调一次 AI 把它整理成干净 JSON
- 章节大纲 / 卷大纲 / 细纲场景解析：均为「JSON 优先 → AI 重构」，任意书写格式（含 `**标题**摘要` 无冒号）都能正确识别
- 灵感反推、信仰拆分、伏笔结构化等所有"自由文本 → 结构化"环节，统一用 AI 解析，字段名严格对齐存储字段，杜绝脆弱映射

### 16.6 AI 结构化输出展示优化

`AIStreamOutput` 自动检测结构化（JSON）输出：
- 生成中：显示「AI 正在生成结构化内容…完成后点采纳自动整理」，不直接糊原始 JSON
- 完成后：显示「✓ 已生成，点采纳自动填入对应栏目」+ 可折叠「查看原始数据」
- 散文类输出（章节正文等）不受影响，照常流式展示

## 十七、数据安全体系

### 17.1 数据存储

- **主存储**：浏览器 IndexedDB（36 张表，通过 Dexie.js 管理）
- **配置存储**：localStorage（API Key、题材包选择、UI 偏好）
- **不经过任何服务器**：AI 调用直接从浏览器发到 AI 平台 API

### 17.2 数据不丢失保证

| 场景 | 影响 | 保护机制 |
|---|---|---|
| 关闭浏览器 | 无影响 | IndexedDB 持久化 |
| 代码更新（git pull） | 无影响 | 代码和数据物理隔离 |
| DB Schema 升级 | 自动迁移 | Dexie 版本管理，所有升级只加不删 |
| 清除浏览器数据 | **数据丢失** | 需提前导出 JSON 或绑定本地文件夹 |
| 浏览器崩溃 | 可能丢失未保存编辑 | 自动保存 hook（编辑器 3 秒防抖保存） |

### 17.3 备份体系（四层）

1. **自动快照**：每 5 分钟，最多 20 个/项目
2. **手动快照**：随时创建标记点
3. **JSON 导出**：完整 36 表数据，可跨设备恢复
4. **本地文件夹同步**：每次保存自动写入磁盘（Chrome/Edge）
5. **GitHub Gist**：云端备份，有版本历史

### 17.4 导出/导入完整性

JSON 导出/导入覆盖全部 36 张数据表，包括：
- 基础数据：项目、世界观、故事核心、力量体系
- 角色数据：角色、势力、关系
- 创作数据：大纲节点、章节、细纲、情感节拍、故事线、伏笔、状态卡、便签
- 世界数据：地理、历史、道具、世界节点、历史事件、历史关键词
- 参考数据：参考书目、参考分析结果
- 学习数据：作品、分块分析、章节节奏、风格量化、跨作品洞察
- 创作规则

导入时自动重建所有外键关系（大纲→细纲、章节→情感节拍、参考→分析结果等）。

### 17.5 项目删除

删除项目时，级联删除所有关联数据（30+ 张表），先收集子表外键 ID，在事务中先删子表再删主表，确保不留孤儿数据。

## 十八、功能联动全景图

以下是从"开始写一本小说"到"完成一本小说"的完整数据流转：

```
① 创建项目（选流派+模式）
     ↓
② 填写世界观（起源→自然→人文）← AI 逐字段辅助
     ↓ 注入
③ 填写故事设计（主题/冲突/主线）← AI 读取世界观
     ↓ 注入
④ 创建角色（选 role → 填信息）← AI 感知阵容缺口
     ↓ 注入
⑤ 设置创作规则（风格/禁区/参考书）
     ↓ 始终注入所有 AI 调用
⑥ AI 生成卷级大纲 ← 读取②③④⑤
     ↓
⑦ AI 生成章节大纲 ← 读取⑥+上下文
     ↓
⑧ AI 生成细纲（场景拆分）← 读取⑦
     ↓
⑨ AI 生成章节正文 ← 读取②③④⑤⑥⑦⑧+伏笔+状态表+历史
     ↓
⑩ 润色/扩写/去AI味 ← 读取⑤
     ↓
⑪ AI 提取状态变更 → 更新状态表 → ⑨的后续章节可读取
     ↓
⑫ 伏笔管理 ← AI 建议伏笔点 → 注入⑨
     ↓
⑬ 导出（JSON/Markdown/TXT/HTML/EPUB/Gist）
```

**关键的"看不见的线"**：
- 创作规则（⑤）是全局广播——所有 AI 调用都读它
- 状态表（⑪）形成反馈循环——章节生成→提取变更→更新状态→下一章生成时读取新状态
- 伏笔（⑫）在正文生成中被感知——AI 知道当前章节需要埋设哪些伏笔
- 参考作品分析和大师洞察通过创作规则中的勾选进入写作流
- 历史年表 + 真实与幻想规则被注入大纲、正文与场景考证生成
- 多世界模式下，以上所有上下文均按"当前世界"隔离注入（见第十九章）
- 下游产物（物品栏/故事年表/状态表）由 AI 从已写正文回提，可反哺后续写作

## 十九、多世界系统

> Phase 25.4 / 25.5。支持诸天流、无限流、快穿、修仙多界、平行世界等"一部小说多个世界"的题材。

### 19.1 开关与定位

项目概况页新增「🌐 多世界模式」开关，**默认关闭**——关闭时整个工具与单世界时完全一致，无任何多余 UI，90% 单世界用户零感知。开启后：
- 自动把现有项目级的世界观/力量/地理/历史数据归属到「主世界」
- 侧栏设定库出现「🌐 世界总览」入口
- 世界观三面板 / 力量体系 / 地理 / 世界地图 / 历史年表 顶部出现世界切换器
- 角色面板出现世界过滤器，大纲卷节点出现世界标签

### 19.2 数据模型

- `worldGroups` 表：每个世界组（主世界/穿越目标/副本/平行/上界/自定义）有名称、类型、图标、穿越规则（进入条件、能力限制、可带走物品）、预计章节数
- `worldGroupLinks` 表：世界之间的关系（传送门/飞升通道/召唤/分支/回归）
- 现有表新增可选 `worldGroupId`：Worldview / PowerSystem / Geography / History / WorldNode / OutlineNode；Character 另有 `homeWorldGroupId` + `isCrossWorld`
- **数据隔离原则**：每个世界拥有独立的世界观、力量体系、地理、历史、角色；故事核心是项目级（整体主线不分世界）

### 19.3 世界总览面板

- 世界列表（增删改排序）、世界详情编辑（基础信息 + 穿越规则）
- **AI 建议世界**：输入整体故事概念，AI 设计 2-4 个差异化世界，勾选采纳
- **AI 扩写世界观**：给某世界一句话草稿，AI 参考其他世界扩展为完整世界观（避免雷同），直接写入该世界
- **世界关系流向图**：纯 SVG，4 种自适应布局（横向流程=诸天流 / 中心辐射=无限流 / 纵向阶梯=修仙多界 / 树状分支=平行世界），智能默认 + 手动切换；连线按关系类型区分颜色样式
- 穿越总览表格

### 19.4 AI 上下文按世界隔离

- 两个核心上下文函数：`buildCurrentWorldContext`（当前世界完整设定）/ `buildAllWorldsOverview`（所有世界精简摘要）
- 在某世界内生成世界观/章节/地图时，AI 只读该世界设定，互不混淆
- 大纲按卷的世界标签注入对应世界上下文（卷可属于非当前活跃世界）
- 章节编辑器沿大纲父链找到所属卷的世界，注入该世界上下文 + 按世界过滤角色

### 19.5 多世界版灵感反推

灵感反推在多世界模式下输出多个世界框架（区别于"AI 建议世界"：此处用户先给带多世界意图的灵感，AI 顺思路反推）。采纳时批量创建世界组 + 各世界世界观 + 故事核心 + 角色归属。字段严格对齐 v3 世界观字段，确保正确落框。

> **导入导出**：JSON v3 完整保留世界组、世界关系及所有 `worldGroupId` 关联，导入时正确重映射 ID。

## 二十一、下游提取产物（物品栏 / 故事进程年表）

> Phase 25.5.2。一类共同模式：「已写正文 → AI 提取 → 结构化产物 → 展示」。区别于写作前的规划工具，这是写作的**下游产物**。

### 21.1 物品栏（创作区 → 物品栏）

游戏包裹式的物品追踪。

- 聚合视图：每件物品显示**当前持有数量** + 完整的获得/消耗历程（时间线，按章节排序）
- 重新提取时按章先清旧记录，避免重复累加
- 支持手动增删改
- 项目级（主角携带物品跨世界，不按世界组隔离）

### 21.2 故事进程年表（创作区 → 故事年表）

正文里发生过的剧情大事的时间轴。

- 一键「从正文提取年表」：AI 逐章提取剧情大事，带**故事内时间**（如"开元三年春"）+ **重要度分级**（次要/重要/关键）
- 时间轴视图按章节进程排序，重要度色点标识
- 支持手动增删改

> 两者都复用现有的"逐章 AI 提取 + 重提取去重"管线；导入导出完整保留（chapterId 重映射）。

## 二十二、场景考证

> Phase 27.2a。zzjj 需求：构思场景时让 AI 边考证边想符合设定的点子。
