## 1. 设计理念

故事熔炉是一个**帮你管理创作全流程的工坊**。

我们认为，写好一部长篇小说，最难的不是码字本身，而是：
- 世界观越写越乱，前后矛盾
- 角色写到后面就"面目模糊"
- 伏笔埋了忘收，时间线搞混
- AI 生成的文字千篇一律，没有个人风格
- 几十万字的项目数据散落各处，换个浏览器就没了

故事熔炉的设计原则：

- **全链路覆盖** —— 从"脑子里有个想法"到"导出完整小说"，不需要在多个工具之间跳转
- **AI 是助手不是主人** —— 每一步都可以手写，AI 只在你需要时介入，且你可以编辑所有 AI 提示词
- **数据永远在你手里** —— 纯前端运行，所有数据存在你自己的浏览器里，不经过任何服务器
- **模块之间互相"知道"** —— 写章节时，AI 能看到你的世界观、角色、伏笔、情感节拍、参考作品分析——不是各管各的孤岛

## 2. 整体架构一览

打开一个项目后，你会看到左侧的模块导航树。整个工坊分为五大区域：

```
著作信息
├── 项目概况        ← 基本信息、流派、简介
└── 项目参考        ← 参考书目 + 深度分析

设定库
├── 世界观
│   ├── 世界起源
│   ├── 自然环境
│   ├── 人文环境
│   ├── 地理环境    ← 地点列表 + 层级 + AI 地图
│   ├── 历史年表    ← 时间线事件管理
│   └── 世界地图    ← AI 程序化生成 + 多世界树
├── 故事设计        ← 主题、核心冲突、情节模式
├── 道具系统        ← 武器/法宝/丹药/功法等 9 类
└── 角色设计
    ├── 主要角色
    ├── 次要角色
    ├── NPC
    ├── 路人
    └── 关系网      ← 力导向关系图 + 列表

创作区
├── 创作规则        ← 写作风格/禁忌/一致性/引用手法/大师洞察
├── 大纲            ← 卷 → 章 → 节 层级管理
├── 故事线          ← 主线 + 多条支线 + 阶段管理
├── 章节            ← 写作编辑器 + AI 五件套
├── 伏笔            ← 列表/看板双视图 + 四阶段流转
└── 状态表          ← 角色/地点/物品/势力状态追踪

提示词库            ← 所有 AI 提示词的查看/编辑/自定义

设置区
├── 版本历史        ← 快照备份与恢复
├── 导入            ← AI 大文档解析导入
├── 导出            ← JSON/Markdown/TXT/HTML
└── 设置            ← AI 配置 + 主题切换
```

每个模块不是孤立的——它们通过**上下文注入**串联在一起。后面会详细讲。

## 3. 世界观构建

### 3.1 三层世界观

故事熔炉把世界观拆成三个维度，每个维度都有独立的编辑面板：

| 维度 | 内容 | 你能填什么 |
|------|------|-----------|
| **世界起源** | 这个世界是怎么来的 | 创世神话、基本法则、时间起点 |
| **自然环境** | 世界的物理面貌 | 地理、气候、生态、资源分布 |
| **人文环境** | 世界里的"人" | 种族、文明、社会制度、文化、宗教 |

每个维度都支持 **AI 生成建议**——你给一个大致方向，AI 帮你扩写出详细设定，你可以接受、修改或重新生成。

### 3.2 力量体系

独立的力量体系编辑器：名称、描述、等级划分、修炼规则。写修仙/玄幻/科幻的同学尤其需要这个——确保全文修炼体系前后一致。

### 3.3 地理环境（层级地点系统）

- **地点列表** —— 支持 10 种地点类型（大陆/国家/城市/门派/秘境/遗迹/战场/自然景观/建筑/其他）
- **层级关系** —— 每个地点可以设置"上级地点"，形成 大陆 → 国家 → 城市 的树形结构
- **树状图视图** —— 自动根据层级关系渲染可视化树状地图
- **AI 概念地图** —— 一键让 AI 根据地点数据生成 SVG 概念地图
- **图像生成 Prompt** —— 自动生成适合 Midjourney/DALL-E 的地图绘制提示词，复制即用

### 3.4 历史年表

时间线管理器，每个历史事件包含：
- 所属纪元（如"上古纪"）
- 具体时间
- 事件描述
- 对世界的影响

支持手动排序，确保你的世界历史时间线清晰有序。

### 3.5 世界地图（程序化生成）

这是故事熔炉最炫酷的功能之一：

- **Voronoi 程序化地图** —— 基于 Perlin 噪声的地形生成，自动模拟海拔、气候、降水、生物群落
- **河流系统** —— 自动生成合理的河流走向
- **势力领地** —— 根据势力数据自动划分领土边界
- **多世界树** —— 如果你的小说有多个位面/世界，可以建立世界树层级，每个世界独立的地图配置
- **传送门系统** —— 在世界之间标记传送通道
- **标记编辑** —— 在地图上放置自定义标记点

### 3.6 道具系统

9 种道具分类：武器、防具、法宝、丹药、材料、功法秘籍、阵法、特殊物品、其他。

每个道具可记录：名称、类型、品级、描述、能力效果、来历、当前持有者、剧情重要性。

## 4. 角色设计系统

### 4.1 四级角色分类

| 分类 | 说明 | 使用场景 |
|------|------|---------|
| **主要角色** | 主角、主要配角、核心反派 | 需要完整人物小传 |
| **次要角色** | 有名有姓但戏份有限的角色 | 简要描述即可 |
| **NPC** | 功能性角色（店小二、守卫等） | 只需名字和特征 |
| **路人** | 背景角色模板 | 场景填充用 |

每个角色都支持：
- **AI 生成** —— 给出角色定位，AI 帮你生成完整人物设定（外貌、性格、动机、背景）
- **首次出场/退场章节** —— 设置角色活跃范围，AI 写作时自动只带入当前章节活跃的角色
- **角色弧光** —— 记录角色的成长变化

### 4.2 角色关系网

这是一个交互式的力导向关系图：

- **10 种关系类型** —— 亲属、恋人、朋友、对手、敌人、师父、弟子、盟友、上下级、其他
- **双向/单向** —— 支持双向关系（互为朋友）和单向关系（暗恋）
- **可视化** —— 力导向图自动布局，不同角色类型用不同颜色的节点
- **列表视图** —— 在图和列表之间自由切换

### 4.3 势力管理

门派、国家、组织等势力的管理面板，每个势力包含 6 个字段的详细编辑。

## 5. 故事设计与大纲

### 5.1 故事设计（故事核心）

四个核心字段，定义你的故事骨架：
- **主题** —— 这个故事想表达什么
- **核心冲突** —— 推动故事前进的根本矛盾
- **情节模式** —— 升级流/逆袭流/成长流/群像流等
- **故事线** —— 主线剧情概要

### 5.2 大纲管理

层级化大纲系统：**卷 → 章 → 节**

- 树形结构拖拽管理
- 每个节点可编辑标题、摘要
- AI 可根据故事核心和世界观自动生成大纲
- 支持细纲模式——每一章拆分为多个场景，标注节奏（慢/中/快/高潮）、情绪弧线、预估字数

### 5.3 故事线管理

独立于大纲的故事线系统：
- **主线** + 多条**支线**
- 每条线分多个**阶段**，每个阶段有关键事件
- AI 可根据世界观和大纲自动生成故事线
- 写作时自动注入当前章节涉及的故事线上下文

## 6. 章节写作与编辑

这是故事熔炉的核心——写作编辑器。

### 6.1 AI 写作五件套

| 功能 | 说明 |
|------|------|
| **生成** | 根据大纲节点，AI 生成整章内容 |
| **续写** | 从当前文本末尾继续写 |
| **扩写** | 选中一段文字，AI 扩充细节和描写 |
| **润色** | 选中一段文字，AI 优化文笔和表达 |
| **去 AI 味** | 选中一段文字，AI 去除机器感，让文字更自然 |

还有一个**自定义指令**输入框，你可以输入任何指令（如"把这段对话改成方言"、"加入更多环境描写"）。

### 6.2 AI 写作时"知道"什么？

**这是故事熔炉最核心的设计。** 当 AI 帮你写一章内容时，它不是在真空中写作——它能看到：

```
┌─────────────────────────────────────────┐
│           AI 写作时的上下文              │
├─────────────────────────────────────────┤
│ 工作记忆（Working Memory）              │
│  · 当前章节大纲节点（标题 + 摘要）       │
│  · 当前章节的情感节拍卡                  │
│                                         │
│ 情节记忆（Episodic Memory）             │
│  · 状态表：角色/地点/物品当前状态        │
│  · 前一章末尾 500 字（衔接用）           │
│                                         │
│ 语义记忆（Semantic Memory）             │
│  · 世界观摘要（起源+自然+人文）          │
│  · 力量体系                             │
│  · 当前活跃角色列表                      │
│  · 未回收的伏笔                          │
│  · 当前涉及的故事线                      │
│                                         │
│ 额外注入                                │
│  · 题材约束（玄幻/言情/悬疑等流派规则）  │
│  · 写作风格（金庸式/古龙式/张爱玲式等）  │
│  · 引用手法（参考作品的分析方法论）       │
│  · 大师洞察（跨作品归纳的创作规律）       │
│  · 创作规则（禁忌 + 一致性约束）          │
└─────────────────────────────────────────┘
```

**三层记忆系统**会根据当前任务类型（生成/续写/润色）自动调整各层的 token 预算分配，确保最相关的上下文优先注入、不会超出模型上下文窗口。

### 6.3 自动功能

写完一章后，系统会自动：
- **提取状态变化** —— AI 分析文本，检测角色/地点/物品的状态是否发生变化，弹窗让你确认
- **生成章节摘要** —— 自动总结本章要点，供后续章节参考
- **统计字数** —— 实时字数显示

### 6.4 辅助面板

编辑器右侧可以展开多个辅助面板：
- **情感节拍卡** —— 为当前章节规划情绪弧线（节拍标签、场景目标、情绪基调、读者感受、角色成长）
- **审阅面板** —— 文本可读性分析和改进建议
- **便签** —— 5 种颜色的便签，可以绑定到当前章节或设为全局，支持置顶

## 7. 伏笔追踪系统

长篇小说最怕的就是"埋了伏笔忘了收"。故事熔炉提供了一套完整的伏笔管理方案。

### 7.1 十种伏笔类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 🔫 契诃夫之枪 | 出现的物品必须在后续发挥作用 | 墙上挂的剑，后来用于决斗 |
| 🔮 预言暗示 | 对未来事件的隐晦预告 | 老人的一句谶语 |
| 🎭 象征伏笔 | 用象征物暗示主题或命运 | 反复出现的枯树 |
| 👤 角色伏笔 | 角色的隐藏身份或秘密 | 看似普通的老者其实是大能 |
| 💬 对话伏笔 | 对话中不经意的关键信息 | "我从不说谎"（其实在说谎） |
| 🌿 环境伏笔 | 环境描写暗示后续事件 | 天边的红光预示灾变 |
| ⏰ 时间线伏笔 | 时间点或期限的设定 | "三年之约" |
| 🐟 红鲱鱼 | 故意误导读者的假线索 | 让读者怀疑错误的凶手 |
| 🔄 平行伏笔 | 平行叙事中的呼应 | A 线和 B 线在不同时空的同一把钥匙 |
| ↩️ 回调伏笔 | 对前文内容的有趣回调 | 重复台词但语境完全不同 |

### 7.2 四阶段状态流转

```
📋 计划中 → 🌱 已埋设 → 🔔 已呼应 → ✅ 已回收
```

每个伏笔可以关联到具体章节：
- **埋设章节** —— 在哪一章埋下伏笔
- **呼应章节**（可多选） —— 在哪些章节中暗示/呼应了这个伏笔
- **回收章节** —— 在哪一章正式揭晓

### 7.3 双视图

- **列表视图** —— 按状态筛选，点击编辑详情
- **看板视图** —— 四列看板（计划中/已埋设/已呼应/已回收），一目了然

### 7.4 AI 伏笔建议

AI 会根据你的世界观、角色、已有伏笔，建议新的伏笔设计。你可以自定义提示词参数来控制建议方向。

### 7.5 与写作的联动

**写章节时，AI 会看到所有未回收的伏笔。** 这意味着 AI 在写作时会自然地呼应已埋设的伏笔，而不会无视它们。

## 8. 状态表与情感节拍

### 8.1 状态表

追踪小说中所有实体的"当前状态"——随着剧情推进，状态会变化。

**五类追踪对象**：角色、地点、物品、势力、事件

**工作流**：
1. 写完一章 → 系统自动调用 AI 分析文本
2. AI 检测到状态变化（如"主角从练气期突破到筑基期"）
3. 弹出确认窗口，显示变化对比（旧值 → 新值）
4. 你确认后，状态表自动更新

下次 AI 写作时，它会看到最新的状态表，确保不会写出"已经突破筑基的主角还在练气期挣扎"这种矛盾。

### 8.2 情感节拍卡

为每一章规划情绪起伏：

- **节拍标签** —— 如"初遇""误会""和解"
- **场景目标** —— 这个节拍要达成什么
- **情绪基调** —— 紧张/轻松/悲伤/热血/温馨……
- **读者感受** —— 你希望读者在这里感受到什么
- **角色成长** —— 角色在这个节拍中的变化

可以手动编辑，也可以让 AI 根据章节大纲自动生成。写作时，情感节拍会注入 AI 上下文，让生成的文字更贴合你设计的情绪节奏。

## 9. 创作规则与约束

创作规则面板是控制 AI 写作行为的"总指挥部"。

### 9.1 基础规则

| 字段 | 说明 | 示例 |
|------|------|------|
| **写作风格** | AI 的文字风格基调 | "简洁凌厉、不用华丽辞藻" |
| **叙事视角** | 第一人称/第三人称有限/全知/多视角 | 四选一 |
| **基调和氛围** | 作品整体氛围 | "黑暗压抑，偶有温情" |
| **特殊创作要求** | 任何额外要求 | "每章末尾必须有悬念" |

以上三个文本字段都支持 **AI 建议**——AI 会根据你的世界观和故事核心生成建议。

### 9.2 禁忌与一致性

- **禁止事项** —— 列表式管理，如"不能出现现代用语"、"主角不能杀无辜之人"等
- **一致性规则** —— 如"修炼体系必须遵循金木水火土五行"、"同一招式名称前后统一"等

这些规则会在 AI 写作时注入上下文，从系统层面约束 AI 的行为。

### 9.3 引用手法

如果你在「项目参考」中上传了参考作品并完成了深度分析，这里可以**勾选**你想让 AI 参考的作品。

勾选后，AI 写作时会看到这些作品的五维分析结论（世界观范式、角色设计手法、情节节奏规律、伏笔技巧、文笔特征），并参考它们来指导创作。

### 9.4 大师洞察

如果你在「作品学习 → 手法洞察」中归纳了跨作品的方法论，这里可以**勾选**想要注入的洞察。

勾选后，AI 写作时会看到这些方法论要点，帮助你的创作融入大师级的叙事技巧。

**引用手法 + 大师洞察，就是 FSS 同学问的"八维分析怎么喂给写手"的答案。**

## 10. 作品学习系统

这是故事熔炉最独特的功能之一——**向大师学习**。

### 10.1 设计思路

写小说最好的老师就是优秀的小说。故事熔炉让你上传你喜欢的作品，AI 从五个维度拆解它的创作方法论，然后你可以把这些方法论注入到自己的创作中。

### 10.2 三层分析架构

**Layer 1 — 五维分块分析**

上传一本小说后，系统会把原文切成多块，逐块调用 AI 分析以下五个维度：

| 维度 | 分析什么 |
|------|---------|
| **世界观范式** | 作者是如何搭建/扩展世界观的？"先给规则再立反例"还是"让主角走一遍再解释"？ |
| **角色设计手法** | 角色是怎么立起来的？外貌/台词/动作/内心/反差/偏执？ |
| **情节节奏规律** | 开局→转折→高潮→钩子的字数分配？爽点/虐点切换频率？ |
| **伏笔与悬念** | 埋了哪些伏笔？明线还是暗线？用"揭晓真相"还是"渐进解密"？ |
| **文笔与语言** | 句子长度偏好、描写/对话占比、常用修辞、可模仿的金句模板 |

分析深度可选：**快速**（大块粗览）/ **标准**（适中）/ **深度**（细块精析）

**Layer 2 — 风格量化 + 章节节奏**

- **风格量化画像**（纯本地计算，不消耗 AI）
  - 平均句长 + 句长分布直方图
  - 对话占比（引号字符 / 总字符）
  - 高频词 Top 50（去停用词的 2-4 字词频）
  - 段落密度（每千字多少段）
- **章节节奏时间线**（AI 驱动）
  - 自动切分章节（识别"第X章/回/节"标题）
  - 每章提取 3-8 个关键节奏点
  - 7 种节奏类型：开场/冲突/反转/高潮/章末钩子/伏笔/松弛
  - 可视化时间线，直观看到每章的情绪起伏

**Layer 3 — 跨作品洞察归纳**

当你学习了多本同类作品后，AI 会从它们的分析结果中归纳出**共性的创作方法论**。

比如：
- "猫腻式悬念递进法"
- "爽文开场三板斧"
- "爽虐交替的黄金比例"

每条洞察包含：标题、详细说明、3-5 条可操作要点、适用流派、来源作品。

### 10.3 完整流程

```
上传 .txt/.docx/.pdf
        ↓
  Layer 1 五维分析（AI，逐块）
        ↓
  Layer 2 风格量化（本地）+ 节奏提取（AI，逐章）
        ↓
  分析报告（五维报告 + 风格卡片 + 节奏时间线）
        ↓
  多本作品 → Layer 3 跨作品洞察归纳
        ↓
  在「创作规则」中勾选引用 → 注入 AI 写作上下文
```

### 10.4 数据安全

- 原文 Blob 持久化在 IndexedDB，关闭浏览器后可续跑分析
- 分析结果永久保留在本地
- 支持 ZIP 打包下载完整分析档案
- 整个系统与你的创作数据物理隔离，互不污染

## 12. 数据管理与备份

### 12.1 导出格式

| 格式 | 说明 | 适用场景 |
|------|------|---------|
| **JSON** | 完整项目数据（所有表、所有设定、所有章节） | 备份/迁移/恢复 |
| **Markdown** | 按大纲结构导出正文，带元数据 | 分享/编辑 |
| **TXT** | 纯文本，只有正文 | 投稿/发布 |
| **HTML** | 带样式的网页版，包含大纲、角色表、世界观 | 预览/分享 |

### 12.2 导入能力

- **JSON 导入** —— 从备份恢复完整项目
- **大文档 AI 解析导入** —— 上传已有的小说文档（DOCX/DOC/PDF/ODT/RTF），AI 自动识别并提取：
  - 角色设定
  - 世界观信息
  - 大纲结构
  - 章节内容
  - 支持大文件分块处理（50KB-1MB 可配置）
  - 断点续传 + 失败重试
  - 实时进度跟踪 + 活动日志

### 12.3 版本历史

基于快照的版本管理：
- 支持手动创建快照
- 从任意快照恢复
- 带时间戳的快照列表

### 12.4 数据安全

所有数据存储在浏览器 IndexedDB 中：
- **不经过任何服务器** —— 你的创作内容、世界观设定、角色信息都只在你的浏览器里
- **AI API Key 存 localStorage** —— 只在你的浏览器本地，不会被上传
- **导出备份** —— 强烈建议定期导出 JSON 备份，以防清理浏览器数据

## 15. 功能联动全景图

故事熔炉最强大的地方不是某个单独的功能，而是**所有功能都在一个数据空间里互相感知**。

下面这张图展示了模块之间的数据流向：

```
  ┌──────────────────────────────────────────────────┐
  │                  设定库                            │
  │  世界观 ──┐                                       │
  │  力量体系 ─┤                                      │
  │  角色列表 ─┼──→ 语义记忆 ──→ ┌──────────────┐    │
  │  角色关系 ─┤                 │              │    │
  │  势力设定 ─┘                 │   章节编辑器  │    │
  │                              │  （AI 写作） │    │
  │  创作区                      │              │    │
  │  创作规则 ──→ 约束注入 ──→   │              │    │
  │  大纲摘要 ──→ 工作记忆 ──→   │              │    │
  │  故事线   ──→ 语义记忆 ──→   │              │    │
  │  伏笔列表 ──→ 语义记忆 ──→   │              │    │
  │  状态表   ──→ 情节记忆 ──→   │              │    │
  │  情感节拍 ──→ 工作记忆 ──→   │              │    │
  │                              └──────┬───────┘    │
  │                                     │            │
  │  写作风格预设 ─────────────→ 风格注入 ┘           │
  │  题材约束（玄幻/言情/悬疑…）→ 题材注入 ┘          │
  │                                                  │
  │  作品学习系统                                     │
  │  参考作品分析 ──→ 引用手法 ──→ 注入写作上下文      │
  │  大师洞察    ──→ 洞察注入 ──→ 注入写作上下文      │
  │                                                  │
  │  写完一章后 ←──── 自动提取状态变化 → 更新状态表    │
  │             ←──── 自动生成章节摘要 → 供后续章节用  │
  └──────────────────────────────────────────────────┘
```

**一句话总结**：你在设定库里做的每一个设定，都不会白费——它们最终都会被带入 AI 的写作上下文，让 AI 写出与你的世界观、角色、剧情高度一致的内容。
