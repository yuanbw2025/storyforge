# Token 消耗与省钱指南 —— 你的每一分钱都花在刀刃上

> **适用版本**: 2026 年 5 月  
> **阅读时间**: 约 8 分钟  
> **一句话**: 故事熔炉在 Token 透明化、上下文智能裁剪、分析模型独立选择三个层面，帮你把 AI 消耗降到最低。

---

## 为什么要关心 Token？

用 AI 写小说，每一次请求都会消耗 Token。Token 是大语言模型计费的基本单位——你发给 AI 的内容（输入 Token）和 AI 回给你的内容（输出 Token）都会计费。

中文大约 **1 个汉字 ≈ 1.5 个 Token**（各模型有差异，故事熔炉内置的估算就用这个比例）。

这意味着什么？

| 场景 | 字数 | 约等于 Token 数 |
|------|------|----------------|
| 生成一章正文（3000 字） | 3,000 字 | ~4,500 Token |
| 一次章节生成的完整上下文（系统指令 + 世界观 + 角色 + 大纲 + 前文） | 约 8,000-15,000 字 | ~12,000-22,500 Token |
| 用大师研读分析一本 50 万字的小说（标准深度） | 500,000 字 | ~75 万 Token 输入 |
| 分析一本 **125 万字** 的小说 | 1,250,000 字 | ~187 万 Token 输入 |

写一章正文，消耗几千 Token，几分钱，你不会在意。
但如果你用大师研读分析一本百万字长篇，消耗可能飙到数百万 Token——选错模型，一次分析就能花掉好几块钱。

**故事熔炉做了三件事来帮你省钱。**

## 第一件事：Token 全透明——花了多少，一目了然

### 流式生成实时估算

AI 生成文字时是逐字输出的（流式）。在流式生成过程中，故事熔炉会**实时显示当前已生成的 Token 估算值**，让你在等待的过程中就能感知消耗量级。

生成结束后，如果 AI 平台返回了精确的 Token 用量（大多数平台都会返回），估算值会自动替换为**精确值**，分为输入 Token（↑）和输出 Token（↓）两部分。

### 全模块 Token 显示

不只是章节生成——故事熔炉的以下模块**全部会显示 Token 消耗**：

| 模块 | 说明 |
|------|------|
| 章节编辑器 | 生成正文时显示 |
| 审校面板 | AI 审稿 / Anti-AI 去痕迹 / 追读力评估 |
| 大师研读 | 作品分析的每个分块 |
| 地理/世界地图 | AI 生成地理描述 |
| 提示词好坏示例 | 示例生成 |
| 工作流引擎 | 多步工作流的每一步 |
| 情感节拍 | AI 生成场景情绪曲线 |
| 浮动工具栏 | 选中文字后的润色/扩写/缩写/改写/查漏 |

你在故事熔炉里触发的**每一次 AI 调用，消耗都会显示在你眼前**。没有隐藏消耗，没有暗箱操作。

## 第二件事：上下文窗口预算管理——智能裁剪，不浪费一个 Token

### 问题：上下文越长，Token 越贵

写到第 50 章的时候，你的世界观、角色设定、大纲、前文摘要加在一起，可能已经有几万字了。如果全部塞进 AI 的上下文窗口，每次生成都要消耗巨量输入 Token。

但实际上，很多信息对当前这一章的生成并不关键——比如角色 A 的完整背景故事，在角色 A 不出场的章节里是多余的。

### 解决方案：四层分级注入

故事熔炉把所有上下文内容分为四个优先级层：

| 层级 | 内容 | 说明 |
|------|------|------|
| **L0 — 基础指令** | 系统 prompt、风格模板、创作规则 | 必须包含，不可裁剪 |
| **L1 — 核心上下文** | 本章大纲、前文摘要、本章出场角色 | 强相关，优先保留 |
| **L2 — 扩展上下文** | 世界观设定、全局故事线、伏笔上下文 | 重要但可压缩 |
| **L3 — 增强上下文** | 参考作品、风格示例、历史生成记录 | 锦上添花，预算紧张时优先裁剪 |

### 上下文预算进度条

在章节编辑器中，每次触发 AI 生成前，你会看到一个**上下文预算进度条**。它用四种颜色分别标示 L0-L3 各层的 Token 占比，并显示总用量相对于当前模型上下文窗口的百分比。

- **绿色**：充裕，随便用
- **黄色**：接近上限，注意控制
- **红色**：超出窗口限制

### 自动裁剪

如果上下文超出了当前模型的窗口限制，故事熔炉会提示你，并提供**一键自动裁剪**功能——从 L3 层开始，逐层削减低优先级内容，直到总 Token 数回到安全范围内。

你不需要手动去删减世界观或前文。系统帮你做，而且**一定是先砍最不重要的内容**。

## 省钱实战建议

### 日常写作

1. **不需要刻意省**。写一章正文通常消耗几千到一两万 Token，用任何模型都花不了几分钱。
2. **关注上下文预算条**。如果经常显示黄色或红色，说明你塞了太多上下文，可以考虑精简世界观描述或切换到更大窗口的模型。
3. **浮动工具栏的润色/扩写等操作**消耗很小（选中几百字，输入输出各几百 Token），放心用。

### 大师研读

1. **第一件事：去学习设置选个便宜模型**。如果你有 GLM 或 ModelScope 的 API Key，选免费模型最划算。没有的话，DeepSeek Flash 是性价比之王。
2. **分析深度选「快速」可以先试水**。快速模式切块更大（4 万字/块），AI 调用次数少一半以上。先用快速看个大概，觉得需要细看再用标准或深度模式重新分析。
3. **巨型作品（百万字以上）强烈建议用免费模型**。分析质量的差异其实没有价格差异那么大——Flash 模型的五维分析已经足够提炼出有价值的方法论。

### 各 AI 平台的免费额度

| 平台 | 免费额度 | 备注 |
|------|---------|------|
| Gemini | 每分钟 15 次请求，每天 1500 次 | 免费版足够分析好几本书 |
| GLM (智谱) | glm-4-flash 完全免费 | 需注册智谱开放平台 |

## 技术细节（给感兴趣的同学）

### Token 估算原理

- **流式生成中**：中文字符按 1.5 Token/字估算，英文/标点按 0.25 Token/字符估算
- **生成完成后**：如果 API 返回了 `usage` 字段（包含 `prompt_tokens` 和 `completion_tokens`），则用精确值替换估算值
- **上下文预算计算**：对每个上下文段（系统指令、世界观、角色设定等）分别估算 Token 数，再加总

### 上下文分层注入的工作方式

```
┌─────────────────────────────────────────────┐
│ L0 基础指令 (必须)                            │
│   系统 prompt + 风格模板 + 创作规则            │
├─────────────────────────────────────────────┤
│ L1 核心上下文 (强相关)                         │
│   本章大纲 + 前文摘要 + 出场角色              │
├─────────────────────────────────────────────┤
│ L2 扩展上下文 (按需)                          │
│   世界观 + 故事线 + 伏笔 + 货币体系          │
├─────────────────────────────────────────────┤
│ L3 增强上下文 (锦上添花)                      │
│   参考作品 + 好坏示例 + 历史记录              │
└─────────────────────────────────────────────┘

超出窗口时裁剪顺序: L3 → L2 → L1 (L0 不可裁剪)
```

### 大师研读的分块策略

| 深度 | 每块目标字数 | 每块最大输出 Token | 125 万字 → 约切多少块 |
|------|------------|-------------------|---------------------|
| 快速 | 40,000 字 | 4,096 | ~32 块 |
| 标准 | 25,000 字 | 6,144 | ~50 块 |
| 深度 | 15,000 字 | 8,192 | ~84 块 |

块数直接决定 AI 调用次数，也就直接决定费用。快速模式的块数是深度模式的 1/3 左右。

### 分析专用模型的实现方式

故事熔炉维护两套 AI 配置：

1. **全局 AI 配置**：在「系统设置 → AI 配置」中管理，用于日常写作、审校等
2. **分析专用配置**：在「大师研读 → 学习设置」中管理，仅用于作品分析流水线

分析流水线在调用 AI 时，会优先读取分析专用配置。如果未设置专用模型（选的是"跟随全局设置"），则退化为使用全局配置。两套配置共享同一个 provider 的 API Key，你不需要填两次。

## 总结

| 能力 | 解决的问题 | 用户感知 |
|------|-----------|---------|
| Token 全透明 | 不知道花了多少 | 每次 AI 调用都显示消耗 |
| 上下文预算管理 | 上下文太长白白浪费 Token | 分层裁剪，自动控制在窗口内 |
| 分析专用模型 | 批量分析跟着全局贵模型走 | 独立选便宜模型，费用降 90%+ |
| 费用预估 | 分析完才知道花了多少 | 分析前就看到预估价格 |

故事熔炉的目标是：**让你把钱花在真正需要好模型的地方（写正文），而不是花在可以用便宜模型完成的地方（批量分析）。**
